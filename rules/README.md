## üéØ How to Apply These Frameworks

### For New City Pages

1. **Pre-Writing:**
   - Review `holistic-seo-review-framework.md` sections 1-3
   - Identify primary entities (city, DJ companies, venues)
   - Map entity attributes and values
   - Research 3-5 competitor pages
   - Create question list (What, How, Why, When, Where, Who)

2. **Writing:**
   - Follow H2-as-questions structure
   - Provide 30-40 word extractive answers
   - Include comparison table with 8-12 attributes
   - Write with certainty (no "should," "will," etc.)
   - Apply `blog_anti_ai_framework.json` rules throughout
   - Keep intro short (1-2 paragraphs)

3. **Post-Writing:**
   - Add Schema.org markup (Service, LocalBusiness, FAQPage)
   - Add ~3 contextual internal links
   - Validate schemas with Google Rich Results Test
   - Run through `holistic-seo-audit-process.md` checklist

### For Blog Content

1. **Planning:**
   - Define ONE macro context
   - Avoid self-referential intros
   - Plan sharp, specific subheadings

2. **Writing:**
   - Apply all 27 `blog_anti_ai_framework.json` rules
   - Use real sources and data
   - Vary paragraph length
   - No academic transitions

3. **Editing:**
   - Remove opinion modifiers
   - Check for AI tell phrases
   - Ensure authentic voice
   - End with sharp takeaways

### For Service Pages

1. **Structure:**
   - Entity-Attribute-Value (EAV) framework
   - Clear service attributes (pricing, hours, coverage area)
   - Comparison with alternatives

2. **Content:**
   - Factual, not opinion-based
   - Specific measurements and data
   - Real customer scenarios
   - Pros AND cons (minimum 3 pros, 2 cons)

3. **Schema:**
   - Service schema
   - AggregateRating (if applicable)
   - LocalBusiness schema
   - FAQPage schema

## üí° Key Principles Summary

1. **Entity-Oriented:** Structure content around entities (DJ companies, cities, venues) with clear attributes and values
2. **User-First:** Write for humans, not search engines
3. **Authentic Voice:** Avoid AI patterns and maintain natural, credible writing
4. **Evidence-Based:** Support all claims with data, measurements, and real sources
5. **Comprehensive:** Answer all user questions (What, How, Why, When, Where, Who)
6. **Structured Data:** Implement proper Schema.org markup for all content types
7. **Topical Authority:** Build interconnected content with semantic relationships
